## Шаг 7: Настройте обработку ошибки 404
**Продолжительность: 4:00**

Отредактируйте `Program.cs`, чтобы ASP.NET Core показывал вашу кастомную страницу 404 для неизвестных маршрутов и кодов статуса.

Найдите часть после `app.MapRazorPages();` и добавьте строки:

```csharp
app.UseStatusCodePagesWithReExecute("/StatusCode404");
app.MapFallbackToPage("/StatusCode404");
```

Ваш `Program.cs` должен выглядеть примерно так:

```csharp
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddRazorPages();

var app = builder.Build();

if (!app.Environment.IsDevelopment())
{
    app.UseExceptionHandler("/Error");
    app.UseHsts();
}

app.UseHttpsRedirection();
app.UseStaticFiles();
app.UseRouting();
app.UseAuthorization();

app.UseStatusCodePagesWithReExecute("/StatusCode404");
app.MapRazorPages();
app.MapFallbackToPage("/StatusCode404");

app.Run();
```

Теперь любой запрос, который не соответствует ни одной странице (например, `/несуществующая`), будет показывать вашу страницу 404.

## Шаг 8: Настройте логирование HTTP‑запросов в консоль и файл
**Продолжительность: 5:00**

В ASP.NET Core есть встроенное промежуточное ПО для логирования HTTP‑запросов, но по умолчанию оно пишет только в консоль. Чтобы сохранять логи в файл, мы используем библиотеку **Serilog** — простой и мощный инструмент.

### 8.1 Установите пакеты Serilog

Выберите способ, который соответствует вашей среде разработки.

#### Способ 1: через терминал (dotnet CLI)

Выполните в терминале (в папке проекта):

```bash
dotnet add package Serilog.AspNetCore
dotnet add package Serilog.Sinks.File
```

- `Serilog.AspNetCore` — интеграция с ASP.NET Core.
- `Serilog.Sinks.File` — запись логов в файл.

#### Способ 2: через Visual Studio (NuGet Package Manager)

1. В **Visual Studio** откройте проект `GreenswampSite`.
2. Нажмите правой кнопкой мыши на проект в **Обозревателе решений** и выберите **Управление пакетами NuGet**.
3. Перейдите на вкладку **Обзор**.
4. Найдите пакет `Serilog.AspNetCore` и установите его.
5. Затем найдите пакет `Serilog.Sinks.File` и установите его.

После установки пакетов убедитесь, что в файле проекта `GreenswampSite.csproj` появились соответствующие ссылки.

### 8.2 Настройте Serilog в `Program.cs`

Замените содержимое `Program.cs` на следующий код:

```csharp
using Serilog;

// Настройка Serilog
Log.Logger = new LoggerConfiguration()
    .ReadFrom.Configuration(builder.Configuration) // читает настройки из appsettings.json
    .Enrich.FromLogContext()
    .WriteTo.Console() // вывод в консоль
    .WriteTo.File("logs/log-.txt", 
        rollingInterval: RollingInterval.Day, // новый файл каждый день
        retainedFileCountLimit: 7)            // хранить 7 дней
    .CreateLogger();

/* ...  */
builder.Host.UseSerilog(); // заменяем встроенный логгер на Serilog

/* ... */

// Добавляем логирование HTTP-запросов
app.UseHttpLogging();

```

### 8.3 Добавьте конфигурацию логирования в `appsettings.json`

Откройте файл `appsettings.json` и добавьте секцию `"Serilog"`:

```json
{
  "Serilog": {
    "MinimumLevel": {
      "Default": "Information",
      "Override": {
        "Microsoft.AspNetCore.HttpLogging.HttpLoggingMiddleware": "Information",
        "Microsoft": "Warning",
        "System": "Warning"
      }
    }
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore.HttpLogging.HttpLoggingMiddleware": "Information"
    }
  },
  "AllowedHosts": "*"
}
```

Теперь логи будут:
- Выводиться в консоль (как раньше).
- Сохраняться в папке `logs` с именем файла `log-20250323.txt` (дата будет меняться).

Каждый HTTP‑запрос будет записан с методом, путём, статус‑кодом и длительностью.

> **Важно:** Папка `logs` создаётся автоматически в корне проекта. Убедитесь, что она не добавлена в систему контроля версий (например, добавьте `logs/` в `.gitignore`).

## Часто задаваемые вопросы

- **Почему я вижу пустую страницу?**  
  Убедитесь, что вы не удалили строку `@RenderBody()` в макете. Также проверьте, что содержимое каждой страницы вставлено правильно.

- **Мои изображения не отображаются.**  
  Проверьте пути – они должны начинаться с `/` (например, `/green-toad-logo.svg`). Убедитесь, что файлы действительно находятся в папке `wwwroot`.

- **Как добавить больше страниц?**  
  Просто создайте новый файл `.cshtml` в папке `Pages` с директивой `@page` и укажите `Layout = "_Layout"`.

- **Можно ли использовать отдельный CSS‑файл?**  
  Да! Поместите его в `wwwroot/css/` и подключите в `<head>` макета `_Layout.cshtml`.

- **Как изменить вывод логов в файл?**  
  Добавьте файловый логгер (например, Serilog или встроенное расширение `AddFile`). См. [документацию по логированию в ASP.NET Core](https://learn.microsoft.com/ru-ru/aspnet/core/fundamentals/logging/).
